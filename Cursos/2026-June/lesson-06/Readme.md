# Pipielines

* CD a CI
  * CI : Continuous Integration
    * Whenever a Deverloper commits changes to the repository a CI pipeline gets executed
    * That pipeline just build the source code and executes the unit tests
    * It makes sure thouse new changes does not brake latest source code version
  * CD : Continue Deployment
    * Make new small version of the product frecuentry
    * Its a good practice
